## Technical Specifications

### Consolidated Dataset Schema

```sql
scout_candidates (
  id UUID PRIMARY KEY,
  org_name text,              -- Individual or org name
  org_type text,              -- 'individual' or 'organization'
  website text,               -- Primary contact URL
  summary text,               -- 500-1000 char description
  fit_score int,              -- 1-10 scale
  fit_reason text,            -- Why scored at this level
  source_query text,          -- Hunter query that found it
  status text,                -- 'pending', 'profiled', 'outreached'
  meta jsonb                  -- {"hunter": "...", "niche": "...", "src": "..."}
)
```

### Profile Search Output Schema

```sql
lead_profiles (
  id UUID PRIMARY KEY,
  scout_id UUID,              -- FK to scout_candidates
  donor_capacity text,        -- "High", "Medium", "Emerging"
  ministry_impact text,       -- Key accomplishments summary
  giving_channels text[],     -- Optimal contact methods
  ask_range_low numeric,      -- $500, $2000, $5000, etc.
  ask_range_high numeric,
  relationship_strategy text, -- Long-term engagement plan
  risk_flags text[],          -- "New to support", "Stability concerns", etc.
  opportunity_flags text[],   -- "Ministry expansion", "Major transition", etc.
  created_at timestamp
)
```
